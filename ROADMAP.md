# خارطة طريق 12+ شهر لمشروع بَيَان

ملخّص الفترات الزمنية والأهداف الرئيسية لإطلاق مشروع ناجح وانتشاره في العالم العربي.

Q1 (أساسيات وبيئة التطوير)
- تدوير/إبطال مفاتيح API المسربة.
- تأمين الأسرار (Secret Manager / Streamlit Secrets / .env محلي).
- إنشاء حاوية Docker وبدء CI أساسي.
- كتابة اختبارات وحدات أولية وتهيئة linting.

Q2 (جاهزية إنتاج ونشر تجريبي)
- نشر نسخة تجريبية على Cloud Run أو منصة سحابية مناسبة.
- إعداد مراقبة (Sentry) وMetrics.
- قيود مفاتيح API (referrers, IPs).
- إعداد قاعدة بيانات مؤقتة مع سياسة TTL للملفات الصوتية.

Q3 (تحسين المنتج وتوسيع النطاق)
- تحسين تجربة المستخدم والواجهات باللغة العربية.
- اختبارات A/B لمكونات التقييم.
- تحسين تكلفة استدعاءات API وCaching.

Q4 (نمو وتسويق)
- إطلاق حملة تسويق عربي: موقع هبوط، محتوى تعليمي، مسابقات.
- شراكات مع مؤسسات لغوية وجامعات.
- خطة اشتراكات أو نماذج دخل (B2B للمؤسسات التعليمية).

بعد السنة
- التوسع في لغات وموديلات أكثر تخصصاً.
- إطلاق API للاستخدام التجاري مع توثيق ومفاتيح مُدارة.
